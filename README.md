# AutoDoc Documentation Generator

This script is designed to automatically generate comprehensive documentation for a specified set of source code files using a local Large Language Model (LLM) accessed via Ollama.

## Features

*   **Automated Documentation**: Generates high-quality Markdown documentation from the contents of source code files.
*   **LLM Integration**: Utilizes a local LLM (e.g., Gemma 4 running via Ollama) to perform the document generation based on the source code provided.
*   **Repository Cloning**: Automatically clones the target GitHub repository into a local source directory for analysis.
*   **Configuration Driven**: Supports loading configuration settings from a TOML file (`~/.config/autodoc/config.toml`) to customize API endpoints, model context size, and output paths.
*   **Clean Output**: Focuses on gathering relevant file content while excluding common version control artifacts.

## Installation

The recommended method for installation is via the provided setup script, which handles environment setup and dependency installation.

1.  **Run the Installer Script**: Execute the `install.sh` script to set up the environment and install dependencies:
    ```bash
    ./install.sh
    ```

This process sets up an isolated Python virtual environment and installs required packages, including `ollama` and `gitpython`. The executable will be placed in `$HOME/.local/bin/autodoc`.

## Dependencies

The script relies on the following Python libraries:

*   `ollama`: For interacting with the local Ollama API.
*   `gitpython`: For cloning repository sources.
*   `tomllib`: For reading configuration settings from TOML files.
*   `httpx`, `httpcore`: For API communication.
*   Other core libraries: `anyio`, `pydantic`, etc.

## Usage

The script is executed from the command line, specifying the GitHub repository URL and an optional output path.

### Command Syntax

```bash
autodoc [REPO_URL] [OUTPUT_PATH]
```

### Arguments

| Argument | Description | Default Value |
| :--- | :--- | :--- |
| `REPO_URL` | The GitHub repository URL to clone and analyze. | `https://github.com/Perceptronic-Systems/AutoDoc` |
| `OUTPUT_PATH` | The desired output path for the generated Markdown file. | `~/generated_doc.md` |

### Workflow

1.  The script clones the specified repository into a local source directory.
2.  It gathers content from all relevant files, excluding metadata and ignored files (e.g., `.gitignore`, `LICENSE`).
3.  The gathered code content is passed to the configured Ollama LLM with specific instructions for generating documentation.
4.  The resulting documentation is saved to the specified output file.

## Configuration

Custom configuration settings can be managed in the global TOML configuration file, located at:

`~/.config/autodoc/config.toml`

### Example Configuration (`config.toml`)

```toml
[ollama]
host_address = "http://127.0.0.1:11434"
model_context = 32768

[repository]
link = ""
repo_dir = "~/.cache/autodoc/source"

[output]
output_dir = "~/generated_doc.md"
```

*   **`ollama.host_address`**: Specifies the base URL for the Ollama API endpoint.
*   **`ollama.model_context`**: Sets the context window size passed to the LLM (in tokens).
*   **`repository.repo_dir`**: Defines the local directory where the source code will be cloned.
*   **`output.output_dir`**: Specifies the location where the final generated documentation (`generated_doc.md`) will be saved.

## Execution Details

The core process involves:

1.  **Setup**: Cloning the repository and preparing the source files.
2.  **Gathering**: Recursively reading file contents from the source directory, filtering out ignored files.
3.  **Prompt Construction**: Concatenating all gathered code content into a single string for LLM input.
4.  **Generation**: Sending the combined prompt and source code to the Ollama service for documentation generation.
5.  **Output**: Writing the streamed response to the final output file (`generated_doc.md`).

The interaction with the local LLM occurs via the `ollama.Client` initialized using the configured API host.
